# ТЗ: Сервис проектов (внутренний план по этапам)

Новый сервис — отдельная часть приложения на отдельной странице. Конфигуратор не трогаем.

---

## Этап 1: Роутинг и оболочка ✅
- Установить React Router.
- Конфигуратор остаётся на `/` (главная).
- Новый сервис — на `/projects`.
- Общая оболочка: логотип, навигация (ссылка «Configurator», «Projects»), стили по дизайн-системе.
- После этапа: с главной можно перейти в конфиг и на страницу проектов.

---

## Этап 2: Макеты страниц сервиса проектов (без бэкенда) ✅
- Страница `/projects`: заглушка или список проектов (мок).
- Макет таймлайна: блоки этапов с названием, датами, статусом (мок).
- Макет ленты: блок комментариев под этапом (мок).
- Всё по дизайн-системе (тёмная тема, Almarena, обводки, кнопки).

---

## Этап 3: Бэкенд и модели данных
- Node.js + Express, MongoDB.
- Модели: User (admin/designer/client), Project, Stage, Comment, при необходимости Notification.
- API: регистрация/логин, CRUD проектов, этапов, комментариев.
- Локальный запуск (localhost).

---

## Этап 4: Аутентификация и роли
- JWT, bcrypt для паролей.
- Роли: администратор, дизайнер, заказчик.
- Защита роутов API и фронта по ролям.
- Страницы: логин, после входа — редирект на `/projects` или конфиг.

---

## Этап 5: Таймлайн и этапы (рабочий функционал)
- Отображение этапов с датами и статусами из API.
- Для админа/дизайнера: создание и редактирование этапов, назначение исполнителей.
- Прокрутка по этапам, прикреплённые файлы/версии (базово).

---

## Этап 6: Лента и комментарии
- Лента по этапу/итерации.
- Добавление комментариев, ответы.
- Прикрепление файлов (загрузка, отображение).
- Уведомления (email, Telegram) — по возможности в рамках этапа или отдельной задачей.

---

## Этап 7: Уведомления и деплой
- Email (Nodemailer), Telegram/Slack по API.
- События: новый этап, новый комментарий, этап завершён.
- Тестирование, деплой (пока локальный сервис — без продакшн-деплоя).

---

Использовать дизайн-систему: [DESIGN_SYSTEM.md](./DESIGN_SYSTEM.md).

После каждого этапа — согласование: «норм или нет».
